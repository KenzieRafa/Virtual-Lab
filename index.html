Materi Pembelajaran

BAB 1

Fondasi Pemrograman Python

Tujuan Pembelajaran

  • Konseptual: Mahasiswa memahami alur kerja dasar sebuah program komputer yang terdiri dari Input ‚Üí Proses ‚Üí Output.
  • Praktis: Mahasiswa mampu menulis dan menjalankan program Python sederhana serta memahami bagaimana data diproses di dalam komputer.

Materi Perkuliahan

1. Konsep Inti: Variabel dan Tipe Data

  • Variabel adalah wadah untuk menyimpan data. Analogi: variabel seperti sebuah kotak berlabel yang berisi nilai tertentu.
  • Dynamic Typing di Python: Tidak perlu mendeklarasikan tipe data secara eksplisit, Python otomatis menentukannya saat program berjalan.
  • Konvensi Penamaan: Menggunakan format snake_case (huruf kecil dengan garis bawah sebagai pemisah), misalnya nama_mahasiswa.

A. Jenis Tipe Data Dasar dalam Python:

  • Integer (int): Bilangan bulat (positif, negatif, atau nol)
  • Floating Point (float): Bilangan pecahan atau desimal
  • String (str): Teks yang ditulis dengan tanda kutip
  • Boolean (bool): Nilai logika dengan dua kemungkinan: benar (True) atau salah (False)

B. Kesalahan umum:

  • Mengakses variabel yang belum didefinisikan (menyebabkan error)
  • Salah penulisan nama variabel karena Python membedakan huruf besar dan kecil (case-sensitive)

2. Interaksi Program: Input, Output, dan Konversi Tipe Data

  • Program harus bisa berkomunikasi dengan pengguna
  • Input: Perintah untuk menerima data dari pengguna
  • Output: Perintah untuk menampilkan hasil
  • Semua data dari input awalnya berupa teks (string). Oleh karena itu, sering kali diperlukan konversi tipe data (casting) untuk mengubah string menjadi angka
  • Output dapat diformat agar lebih jelas, misalnya dengan memisahkan beberapa nilai atau menambahkan teks penjelas

3. Operator Aritmatika dan Aturan Prioritas

  • Operator adalah simbol untuk melakukan operasi matematika seperti penjumlahan, pengurangan, perkalian, dan pembagian
  • Prioritas operasi (presedensi): Aturan yang menentukan urutan eksekusi operasi. Misalnya, operasi dalam tanda kurung dieksekusi lebih dahulu, kemudian pangkat, lalu perkalian/pembagian, baru penjumlahan/pengurangan
  • Catatan penting: di Python, pembagian biasa selalu menghasilkan bilangan desimal, meskipun hasilnya bulat
BAB 2

Mengontrol Alur Program

Tujuan Pembelajaran

  • Konseptual: Mahasiswa memahami bagaimana alur eksekusi program dapat dikontrol agar tidak berjalan lurus saja, melainkan dapat bercabang atau berulang.
  • Praktis: Mahasiswa mampu menerapkan percabangan dan perulangan untuk menyelesaikan permasalahan logika sederhana.

Materi Perkuliahan

1. Percabangan (if-elif-else)

  • Alur keputusan: Program sering kali harus memilih jalur berdasarkan kondisi tertentu. Misalnya, jika nilai ujian ‚â• 85, maka hasilnya "A".
  • Indentasi: Python menggunakan indentasi (spasi menjorok ke dalam) untuk menandai blok kode. Kesalahan indentasi akan membuat program tidak dapat dijalankan.
  • Truthy dan Falsy: Selain True dan False, Python juga menafsirkan nilai lain sebagai kondisi benar atau salah. Contoh: angka 0, string kosong, atau list kosong dianggap salah (False).

2. Perulangan (Looping)

  • While Loop: Digunakan ketika jumlah pengulangan belum diketahui. Perulangan berlangsung selama kondisi tertentu masih benar.
  • For Loop: Digunakan untuk mengulang berdasarkan data yang sudah jelas jumlahnya.
  • Break: Menghentikan perulangan sepenuhnya.
  • Continue: Melewati satu iterasi lalu melanjutkan ke iterasi berikutnya.

3. Kesalahan Umum

  • Tidak memperbarui variabel kondisi dalam while loop sehingga menimbulkan loop tak berujung.
  • Salah memahami fungsi range(), karena range(n) menghasilkan urutan dari 0 sampai n-1, bukan sampai n.
BAB 3

Bekerja dengan Kumpulan Data

Tujuan Pembelajaran

  • Konseptual: Mahasiswa memahami perbedaan antara tipe data yang bisa diubah (mutable) dan yang tidak bisa diubah (immutable).
  • Praktis: Mahasiswa mampu menyimpan, mengakses, dan memanipulasi data dalam jumlah banyak menggunakan string dan list.

Materi Perkuliahan

1. String sebagai Sekuens Data

  • String adalah kumpulan karakter yang bersifat immutable. Artinya, setelah string dibuat, isinya tidak dapat diubah.
  • Operasi seperti mengubah huruf menjadi kapital atau menghapus spasi hanya menghasilkan string baru, bukan memodifikasi string asli.
  • String memiliki banyak metode bawaan, seperti mengubah ke huruf besar, menghapus spasi di awal/akhir, mengganti kata tertentu, atau memisahkan string menjadi daftar kata.

2. List sebagai Koleksi Dinamis

  • List adalah kumpulan elemen yang bersifat mutable, artinya bisa ditambah, dihapus, atau diubah.
  • List digunakan untuk menyimpan data dalam jumlah banyak yang dapat berubah-ubah.
  • Python menyediakan metode bawaan untuk list, misalnya menambah elemen, menghapus, menghitung jumlah elemen, dan mengurutkan.
  • Penting untuk memahami bahwa beberapa metode list langsung mengubah data asli dan tidak mengembalikan nilai baru.

3. Kesalahan Umum

  • Menggunakan metode string pada list, atau sebaliknya.
  • Mengira metode sort() menghasilkan list baru, padahal sebenarnya mengubah list yang sudah ada.
BAB 4

Struktur Data Lanjutan dan Pola Perulangan

Tujuan Pembelajaran

  • Konseptual: Mahasiswa memahami bagaimana data dua dimensi dapat direpresentasikan dalam bentuk baris dan kolom.
  • Praktis: Mahasiswa mampu membuat dan mengolah list 2D dengan menggunakan perulangan bersarang.

Materi Perkuliahan

1. Perulangan Bersarang

  • Perulangan bersarang adalah perulangan di dalam perulangan.
  • Analogi: seperti jam, di mana jarum panjang (menit) bergerak penuh setiap kali jarum pendek (jam) bergeser satu langkah. Dengan kata lain, loop dalam harus menyelesaikan seluruh iterasinya untuk setiap satu iterasi loop luar.
  • Nested loop umum digunakan untuk memproses data berbentuk tabel atau grid.

2. List Dua Dimensi (Matriks)

  • List 2D adalah list yang berisi list lain.
  • Data diakses menggunakan indeks ganda: indeks baris dan indeks kolom.
  • Matriks memungkinkan penyimpanan data dalam bentuk tabel, misalnya skor pertandingan atau papan permainan.

3. Kesalahan Umum

  • Mengakses indeks yang tidak ada (IndexError).
  • Membalik urutan indeks baris dan kolom.
BAB 5

Membuat Kode Modular dengan Fungsi

Tujuan Pembelajaran

  • Konseptual: Mahasiswa memahami konsep abstraksi dan dekomposisi masalah dengan membagi program besar menjadi bagian-bagian kecil.
  • Praktis: Mahasiswa mampu membuat fungsi yang rapi, efisien, serta dapat digunakan kembali.

Materi Perkuliahan

1. Abstraksi dan Dekomposisi

  • Dekomposisi: Memecah masalah besar menjadi sub-masalah kecil yang lebih mudah dikelola.
  • Abstraksi: Menyembunyikan detail implementasi dan hanya menampilkan cara penggunaan. Misalnya, saat menggunakan fungsi bawaan seperti print(), kita tidak perlu tahu cara kerjanya, cukup tahu hasil yang diberikan.
  • Fungsi dapat diberi dokumentasi (docstring) untuk menjelaskan tujuan, parameter, dan hasil keluaran.

2. Parameter, Argumen, dan Nilai Kembali (Return)

  • Fungsi dapat menerima data melalui parameter.
  • Argumen dapat diberikan berdasarkan urutan (positional) atau berdasarkan nama (keyword).
  • Parameter juga dapat memiliki nilai default sehingga opsional saat fungsi dipanggil.
  • Fungsi bisa mengembalikan hasil dengan return, yang dapat digunakan dalam perhitungan lebih lanjut.

3. Variabel Lokal dan Global (Scope)

  • Variabel lokal: Dibuat di dalam fungsi dan hanya dapat diakses di fungsi tersebut.
  • Variabel global: Dibuat di luar fungsi dan dapat diakses di mana saja.
  • Sebaiknya hindari mengubah variabel global dari dalam fungsi, karena dapat membingungkan alur program.
